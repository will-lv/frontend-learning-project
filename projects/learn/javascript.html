第1章：JavaScript基础与类型系统

1.1 数据类型详解

JavaScript有8种基本数据类型，理解类型系统是掌握JavaScript的基础。

❓ JavaScript有哪些数据类型？

答案：

基本类型（原始类型）7种：

  • Number：数字类型，包括整数和浮点数
  • String：字符串类型
  • Boolean：布尔类型，true/false
  • undefined：未定义
  • null：空值
  • Symbol：唯一标识符（ES6+）
  • BigInt：大整数类型（ES2020+）

引用类型1种：

  • Object：对象类型，包括普通对象、数组、函数等
// 检测数据类型的方法
console.log(typeof 42);              // "number"
console.log(typeof "Hello");         // "string"
console.log(typeof true);            // "boolean"
console.log(typeof undefined);       // "undefined"
console.log(typeof null);            // "object" (历史遗留bug)
console.log(typeof Symbol());        // "symbol"
console.log(typeof 123n);            // "bigint"
console.log(typeof {});              // "object"
console.log(typeof []);              // "object"
console.log(typeof function(){});    // "function"

// 更准确的类型检测
console.log(Object.prototype.toString.call(null));     // "[object Null]"
console.log(Object.prototype.toString.call([]));       // "[object Array]"
console.log(Array.isArray([]));                        // true

1.2 类型转换与比较

JavaScript的类型转换是面试中的高频考点，包括隐式转换和显式转换。

❓ == 和 === 的区别？

答案：

特性 == (相等) === (严格相等)
类型转换 会进行类型转换 不进行类型转换
比较规则 值相等即可 值和类型都必须相等
性能 较慢（需要转换） 较快（直接比较）
推荐使用 一般不推荐 强烈推荐 
// == 的隐式转换示例
console.log(1 == "1");        // true (字符串转数字)
console.log(true == 1);       // true (布尔转数字)
console.log(null == undefined); // true (特殊规则)
console.log(0 == false);      // true (布尔转数字)
console.log("" == false);     // true (多步转换)

// === 严格比较
console.log(1 === "1");       // false (类型不同)
console.log(true === 1);      // false (类型不同)
console.log(null === undefined); // false (类型不同)
console.log(0 === false);     // false (类型不同)

// 特殊情况
console.log(NaN == NaN);      // false
console.log(NaN === NaN);     // false
console.log(Object.is(NaN, NaN)); // true

console.log(+0 === -0);       // true
console.log(Object.is(+0, -0)); // false

第2章：函数、作用域与闭包

2.1 函数定义与this指向

❓ JavaScript中this的指向规则？

this指向的4种情况：

1. 默认绑定（全局调用）

function foo() {
    console.log(this); // 严格模式下是undefined，非严格模式是window
}
foo(); // 普通函数调用

// 严格模式
"use strict";
function bar() {
    console.log(this); // undefined
}
bar();

2. 隐式绑定（对象方法调用）

const obj = {
    name: '张三',
    greet() {
        console.log(this.name); // '张三'
    }
};
obj.greet(); // this指向obj

// 注意：赋值后丢失绑定
const fn = obj.greet;
fn(); // this指向window(或undefined)

3. 显式绑定（call/apply/bind）

function introduce() {
    console.log(`我是${this.name}`);
}

const person1 = { name: '李四' };
const person2 = { name: '王五' };

// call方法
introduce.call(person1); // '我是李四'
introduce.call(person2); // '我是王五'

// apply方法（参数是数组）
introduce.apply(person1); // '我是李四'

// bind方法（返回新函数）
const boundFn = introduce.bind(person1);
boundFn(); // '我是李四'

4. new绑定（构造函数）

function Person(name) {
    this.name = name;
    console.log(this); // 新创建的对象
}

const p1 = new Person('赵六'); // this指向新对象p1

// 箭头函数没有this
const ArrowFunc = (name) => {
    this.name = name; // this继承自外层作用域
};
// new ArrowFunc('test'); // TypeError

优先级：new > bind > call/apply > 对象方法 > 默认绑定

2.2 闭包详解

❓ 什么是闭包？闭包的作用和注意事项？

闭包定义：

闭包是指有权访问另一个函数作用域中变量的函数。即使外部函数已经返回，内部函数仍然可以访问外部函数的变量。

1. 基本闭包示例：

function outerFunction(x) {
    // 外部函数的变量
    let outerVariable = x;
    
    // 内部函数（闭包）
    function innerFunction(y) {
        console.log(outerVariable + y); // 访问外部变量
    }
    
    return innerFunction;
}

const myClosure = outerFunction(10);
myClosure(5); // 输出：15

// 即使outerFunction执行完毕，innerFunction仍能访问outerVariable

2. 闭包的实际应用：

// 模拟私有变量
function createCounter() {
    let count = 0; // 私有变量
    
    return {
        increment() {
            count++;
            return count;
        },
        decrement() {
            count--;
            return count;
        },
        getCount() {
            return count;
        }
    };
}

const counter = createCounter();
console.log(counter.increment()); // 1
console.log(counter.increment()); // 2
console.log(counter.getCount());  // 2
// console.log(count); // ReferenceError: count未定义

3. 循环中的闭包陷阱：

// 错误示例
for (var i = 0; i < 3; i++) {
    setTimeout(() => {
        console.log(i); // 输出：3 3 3
    }, 1000);
}

// 解决方案1：使用let
for (let i = 0; i < 3; i++) {
    setTimeout(() => {
        console.log(i); // 输出：0 1 2
    }, 1000);
}

// 解决方案2：使用闭包
for (var i = 0; i < 3; i++) {
    (function(j) {
        setTimeout(() => {
            console.log(j); // 输出：0 1 2
        }, 1000);
    })(i);
}

// 解决方案3：使用bind
for (var i = 0; i < 3; i++) {
    setTimeout(function(j) {
        console.log(j); // 输出：0 1 2
    }.bind(null, i), 1000);
}

闭包的优缺点：

  • 优点：
    • 可以实现数据封装和私有变量
    • 可以创建函数工厂
    • 在回调函数中保持状态
  • 缺点：
    • 容易造成内存泄漏（变量无法被垃圾回收）
    • 性能开销（需要维护额外的作用域链）

第3章：对象、原型链与继承

3.1 原型链与继承

❓ JavaScript的原型链是什么？如何实现继承？

原型链概念：

每个JavaScript对象都有一个原型对象，原型对象也可能有自己的原型，这样形成的链条就是原型链。

1. 原型链基础：

// 构造函数
function Person(name) {
    this.name = name;
}

// 在原型上添加方法
Person.prototype.sayHello = function() {
    console.log(`Hello, I'm ${this.name}`);
};

const person1 = new Person('张三');
const person2 = new Person('李四');

person1.sayHello(); // "Hello, I'm 张三"
person2.sayHello(); // "Hello, I'm 李四"

// 原型链查找
console.log(person1.__proto__ === Person.prototype); // true
console.log(Person.prototype.__proto__ === Object.prototype); // true
console.log(Object.prototype.__proto__ === null); // true

2. ES6 Class继承（推荐）：

class Animal {
    constructor(name) {
                        this.name = name;
    }
    
    eat() {
        console.log(`${this.name} is eating`);
    }
}

class Dog extends Animal {
    constructor(name, breed) {
        super(name); // 调用父类构造函数
        this.breed = breed;
    }
    
    bark() {
        console.log(`${this.name} is barking`);
    }
}

const dog1 = new Dog('旺财', '金毛');
dog1.eat();  // "旺财 is eating"
dog1.bark(); // "旺财 is barking"

第6章：🔄 异步编程与事件循环

5.1 事件循环机制

❓ JavaScript事件循环是如何工作的？

事件循环组成部分：

  • 调用栈（Call Stack）：执行同步代码
  • Web APIs：浏览器提供的异步API
  • 任务队列（Task Queue）：宏任务队列
  • 微任务队列（Microtask Queue）：优先级更高

执行顺序：

function debounce(func, delay) {
    let timer = null;
    return function(...args) {
        clearTimeout(timer);
        timer = setTimeout(() => {
            func.apply(this, args);
        }, delay);
    };
}

💡 参考答案：
function deepClone(obj) {
    // 基本类型直接返回
    if (obj === null || typeof obj !== 'object') {
        return obj;
    }
    
    // 处理Date
    if (obj instanceof Date) {
        return new Date(obj);
    }
    
    // 处理Array
    if (obj instanceof Array) {
        return obj.map(item => deepClone(item));
    }
    
    // 处理Object
    const cloned = {};
    for (let key in obj) {
        if (obj.hasOwnProperty(key)) {
            cloned[key] = deepClone(obj[key]);
        }
    }
    return cloned;
}
